# PM Context for AI Assistants

This file provides the **Product Management perspective** on NextPM. Read this to understand goals, users, and success metrics.

## üéØ Product Vision

**Mission**: Empower Product Managers to 10x their productivity using AI tools

**Vision**: NextPM becomes the go-to resource for PMs learning AI-augmented workflows, with:
- Comprehensive workflow documentation
- Reusable prompt library (50+ templates)
- Real case studies with measurable outcomes
- Active community of AI-curious PMs

## üë• Target Users

### Primary Persona: "AI-Curious Mid-Level PM"

**Background:**
- 3-5 years PM experience
- Uses ChatGPT occasionally, wants to do more
- Non-technical background (MBA, business, design)
- Feels pressure to "learn AI" but doesn't know where to start
- Time-constrained (wants practical, not theoretical)

**Goals:**
- Write specs faster without sacrificing quality
- Prototype ideas without waiting for dev resources
- Synthesize user research more efficiently
- "Speak developer" enough to collaborate better

**Pain Points:**
- Generic AI content doesn't apply to PM work
- Technical tutorials assume dev knowledge
- No clear ROI on time investment in AI tools
- Doesn't know which tools to learn first

### Secondary Persona: "Senior PM / Product Leader"

**Background:**
- 7+ years PM experience
- Manages other PMs or leads product area
- Wants team to adopt AI but needs proven patterns
- Budget/time for experimentation

**Goals:**
- Evaluate AI tools for team adoption
- Create standards for AI use in PM work
- Measure productivity gains
- Scale best practices across team

## üé® Product Principles

### 1. **PM-First, Not Dev-Heavy**
- Assume zero coding knowledge
- Explain technical concepts simply
- Focus on PM outcomes, not tech for tech's sake

### 2. **Practical Over Theoretical**
- Every guide includes real examples
- Show actual prompts, not just descriptions
- Measurable outcomes (time saved, quality improved)

### 3. **Honest About Limitations**
- What AI can't do (yet)
- When traditional methods are better
- Realistic time investment expectations

### 4. **Learning in Public**
- Author's journey is transparent
- Mistakes and learnings documented
- Continuous improvement based on feedback

### 5. **Community-Oriented**
- Encourage sharing and remixing
- Give credit to sources
- Build connections, not just content

## üöÄ User Journey

### Discovery
- PM searches "AI for product management" or "ChatGPT for PMs"
- Finds NextPM via search, community share, or social
- Lands on homepage or specific workflow guide

### First Visit
- Reads 1-2 workflow guides (e.g., "Writing Specs with Claude")
- Copies a prompt template to try
- Browses prompt library
- Checks case studies for validation
- **Goal**: "This is practical and relevant to my work"

### Returning Visit
- Has tried prompts, comes back for more
- Explores different workflows
- Compares to their own PM process
- **Goal**: "I'm getting value from this"

### Power User
- Regularly references prompt library
- Tries interactive tools (Phase 2)
- Shares NextPM with colleagues
- Provides feedback or contributions
- **Goal**: "This is part of my PM toolkit"

## üéØ Key Product Decisions

### Why Static Site First?
- Faster to launch, focus on content quality
- Author learning MkDocs is valuable
- Can migrate to dynamic without losing content
- Lower hosting costs during validation

### Why Prompt Library?
- Unique value prop (not just blog posts)
- Reusable, shareable, forkable
- Demonstrates AI-native approach
- Can become standalone product

### Why Dual Documentation (PM + Dev)?
- Author's learning journey is interesting to other PMs
- Shows "PM learning to code with AI" in action
- Helps PMs understand dev perspectives
- Unique angle differentiates from other AI PM content

### Why Azure?
- Author has MSDN subscription (cost-effective)
- Learning Azure is valuable for PM career
- Static Web Apps perfect for this use case
- Prepares for enterprise features (if needed)

## üö´ Out of Scope (For Now)

- Enterprise/B2B focus (individual PMs first)
- Paid content/courses (free tier first)
